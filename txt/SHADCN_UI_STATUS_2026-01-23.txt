## KEY INSIGHTS

### 1. No Direct Shadcn/ui Usage
- Zero NPM packages named `shadcn-ui` or `@shadcn/*`
- Projects copied the shadcn/ui component STYLE, not the package
- Components are custom-built using @radix-ui primitives

### 2. Multiple UI Strategies
- **Intended**: FakeMUI (FakeMUI in workflowui, frontends/nextjs)
- **Inherited**: Custom shadcn-style (codegen, pastebin)
- **Problematic**: @mui/material (postgres)

### 3. @radix-ui is Foundational
- Used in codegen (25 packages)
- Used in pastebin (13+ packages)
- Used in FakeMUI under the hood
- = Standard primitive layer across active projects

### 4. Component Style Convergence
All three approaches use same component patterns:
```typescript
// FakeMUI
import { Button } from '@metabuilder/fakemui'

// Custom shadcn-style in codegen/pastebin
import { Button } from '@/components/ui/button'

// @mui/material (postgres)
import { Button } from '@mui/material'
```
All three are interchangeable in theory, but mixing creates API confusion.
