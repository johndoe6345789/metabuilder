## Notes for Future Reference

### Tailwind 4 API Changes
If upgrading from 3.x to 4.x in the future:
- **Breaking**: Color system changes, new variable syntax
- **CSS**: `@import "tailwindcss"` instead of `@tailwind directives`
- **Plugins**: Some v3 plugins may not work with v4

All current projects already on v4.x, so no migration needed.

### Caret Versioning Rationale
- `^4.1.18` locks major version (4.x), allows minor/patch updates
- Safe for security patches (4.1.x)
- Safe for breaking changes in major version (would require ^5.0.0)
- Recommended approach for stable projects
